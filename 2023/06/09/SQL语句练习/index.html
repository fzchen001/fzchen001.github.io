MySQL语句练习

LeetCode高频 SQL 50 题（基础版）

1. 寻找用户推荐人（NULL值判断）

写一个查询语句，返回一个客户列表，列表中客户的推荐人的编号都 不是 2。

避免错误的秘诀在于使用 IS NULL 或者 IS NOT NULL 两种操作来对 NULL 值做特殊判断。不能使用!=null进行判断。

2. 文章浏览Ⅰ（去重）

请编写一条 SQL 查询以找出所有浏览过自己文章的作者，结果按照 id 升序排列。有重复行！

可以使用distinct来去重重复，使用as来起别名。

3. 无效的推文（字符长度）

写一条 SQL 语句，查询所有无效推文的编号（ID）。当推文内容中的字符数严格大于 15 时，该推文是无效的。

  1. char_length(str)
    （1）计算单位：字符
    （2）不管汉字还是数字或者是字母都算是一个字符
  2. length(str)
    （1）计算单位：字节
    （2）utf8编码：一个汉字三个字节，一个数字或字母一个字节。
    （3）gbk编码：一个汉字两个字节，一个数字或字母一个字节。

4. 使用唯一标识码替换员工ID（左外连接）

左外连接：用于不仅返回满足连接条件的所有记录，而且还返回不满足连接条件的连接操作符左边表的其他行，没有用null值填充。

5. 进店却未进行过交易的顾客

有一些顾客可能光顾了购物中心但没有进行交易。请你编写一个 SQL 查询，来查找这些顾客的 ID ，以及他们只光顾不交易的次数。

  1. 选择了两个表：Visits（访问表）和 Transactions（交易表）。
  2. 通过 LEFT JOIN 连接这两个表，连接条件是 v.visit_id = t.visit_id，这意味着根据访问 ID 将两个表关联起来。
  3. 使用 WHERE 子句过滤条件，amount IS NULL 表示只选择交易表中金额为空的记录。
  4. 使用 GROUP BY 子句按照 customer_id 对结果进行分组，即将具有相同 customer_id 的记录聚合在一起。
  5. 使用 COUNT(*) 函数计算每个顾客的记录数，并将其命名为 count_no_trans。
  6. 最后，返回结果，包括 customer_id 和相应的记录数 count_no_trans。

这段代码的目的是获取在访问表中有记录，但交易表中没有与之关联的交易记录的顾客ID，并统计每个顾客ID对应的记录数量。

6. 上升的温度

编写一个 SQL 查询，来查找与之前（昨天的）日期相比温度更高的所有日期的 id 。

INNER JOIN(等值连接) 只返回两个表中联结字段相等的行。

DATEDIFF(b.recordDate, a.recordDate)两个日期的天数差集，前面的是endDate，后面的的是startDate。

7. 每台机器的进程平均运行时间

请写出一条SQL计算每台机器各自完成一个进程任务的平均耗时.

完成一个进程任务的时间指进程的’end’ 时间戳 减去 ‘start’ 时间戳. 平均耗时通过计算每台机器上所有进程任务的总耗费时间除以机器上的总进程数量获得.

ROUND()四舍五入保留指定的位数。

如果activity_type = "start"是起始时间，将时间戳的负值作为计算的一部分，反之，如果不是start，说明是end，将时间戳的正值作为计算的一部分。

GROUP By machine_id 根据机器id进行分组。

*2，之所以要乘2，是因为如果不乘2，计算的结果就是(endTime - startTime)/2，多除了一倍。

8. 学生们参加各科测试的次数(笛卡尔积，三表联查，分组排序)

要求写一段 SQL 语句，查询出每个学生参加每一门科目测试的次数，结果按 student_id 和 subject_name 排序。

解题思路：首先，先得看最后的表格要求是什么样的。我们观察发现，表格前几列是每个学生都有对应的科目记录行，不管是否参加，所以要把学生表和科目表进行cross join保证每个学生后面都有每个科目的记录；

然后发现，最后一列是求的每个学生，每个科目的参考次数。2个每个，所以groupby 学生，科目；

注意我们应该以第一个cross join的表为主表因为有的学生可能没参加某个科目的考试。
所以我们left join 到exam表格。

最后将两个表合并，重新取数分组，就可以得到结果啦。

  • 先连结两个表：对表Students和表Subjects求笛卡尔积，得到4*3行的表；
  • 再连结三个表：对刚刚得到的新表和表Examinations进行左连结LEFT JOIN；
  • 进行查询。

这里不能用COUNT(*)，要用COUNT(e.subject_name)，因为e.subject_name中才有NULL。COUNT(column)返回不包含NULL值的所有行数。

为什么要使用Subjects表？

主要是存在学生没参加那门课的考试，结果参加次数是0这种情况；没有Subjects这张表，就不可能知道他有什么课没参加。
