栈和队列

Created2023-03-01|算法
|Post View:

理论基础

栈：先入后出，主要可以用来做相邻字符消除的题目

队列：先入先出

java中栈与队列的相关函数：

栈的常用函数 ：

Stack stack = new Stack(Integer);

1、stack.pop(num); // 栈顶元素出栈，并返回栈顶元素。
2、stack.peek(num);//仅返回栈顶元素，并不会将元素弹出。
3、stack.isEmpty();//判断栈是否为空
4、stack.push();//入栈
5、stack.search(); //元素距离栈顶的位置，第几个就返回几，没有返回-1；

队列常用函数：

Queue queue = new LinkedList<>();

1、remove 移除并返回队列头部的元素 如果队列为空，则抛出一个NoSuchElementException异常
2、element 返回队列头部的元素 如果队列为空，则抛出一个NoSuchElementException异常
3、offer 添加一个元素并返回true 如果队列已满，则返回false
4、poll 移除并返问队列头部的元素 如果队列为空，则返回null
5、peek 返回队列头部的元素 ，未移除，与栈中仅返回元素一样 如果队列为空，则返回null
6、add(); 　.put 如果队列满，则阻塞这也是集合所共有的函数；

例题3：有效的括号20

不匹配的情况：

  1. 第一种情况：已经遍历完了字符串，但是栈不为空，说明有相应的左括号没有右括号来匹配，所以return false
  2. 第二种情况：遍历字符串匹配的过程中，发现栈里没有要匹配的字符。所以return false
  3. 第三种情况：遍历字符串匹配的过程中，栈已经为空了，没有匹配的字符了，说明右括号没有找到对应的左括号return false

那么什么时候说明左括号和右括号全都匹配了呢，就是字符串遍历完之后，栈是空的，就说明全都匹配了。

但还有一些技巧，在匹配左括号的时候，右括号先入栈，就只需要比较当前元素和栈顶相不相等就可以了，比左括号先入栈代码实现要简单的多了！

例题6：滑动窗口最大值239（单调队列）

单调队列是一种特殊的队列，它可以在队列中维护一个单调递增或递减的顺序。在使用单调队列时，一般会在队列中存储元素的下标，而不是元素本身。单调队列可以高效地解决一类问题，例如求滑动窗口的最大值或最小值等问题。

在Java中，可以使用双端队列（Deque）来实现单调队列。具体来说，可以将队列中存储的元素按照单调递增或递减的顺序排列。在向队列中添加元素时，如果新元素比队列中的尾部元素更大（或更小），则将尾部元素弹出，直到新元素可以加入队列为止。在删除队列中的元素时，如果队列头部的元素已经不在滑动窗口的范围内，就将其从队列中删除。

特别注意：

单调队列中存储的是元素的下标，而非元素本身。如果要获取元素，需nums[queue.peekLast()]

上述代码中，使用了双端队列来维护一个单调递减的顺序。

  1. 首先，将队列中不在滑动窗口内的元素从队列中删除。
  2. 然后，将队列尾部小于当前元素的元素从队列中删除。
  3. 接着，将当前元素加入队列。如果已经形成了一个滑动窗口，就记录窗口的最大值。
  4. 最后，返回所有滑动窗口的最大值。

例题7：前k个高频元素347（优先队列，堆）

基础：

大顶堆和小顶堆都是二叉堆的一种，是一种基于完全二叉树实现的数据结构。

大顶堆是一种满足父节点值大于或等于子节点值的堆，也就是说堆顶是所有元素的最大值。小顶堆是一种满足父节点值小于或等于子节点值的堆，也就是说堆顶是所有元素的最小值。

在Java中，可以使用PriorityQueue来实现堆的功能。默认情况下，PriorityQueue是小顶堆，但是可以通过传入自定义比较器来实现大顶堆。

在使用堆时，最常见的操作是插入元素和获取堆顶元素。对于小顶堆来说，插入元素时将元素加入堆底，然后向上调整保证堆的性质；获取堆顶元素时直接返回堆顶元素，然后将堆底元素放到堆顶并向下调整保证堆的性质。对于大顶堆来说，插入元素和获取堆顶元素的过程类似，只不过比较器的实现需要做出相应的修改。

堆的应用非常广泛，其中一个典型的例子是Top K问题，即从一个元素序列中找到前K个最大或最小的元素。使用堆可以很好地解决这个问题。

可以使用哈希表和堆的结构来解决这个问题。具体步骤如下：

  1. 使用哈希表来记录每个元素出现的频率，将元素作为键，频率作为值。
  2. 使用一个小顶堆来维护出现频率前k高的元素，堆中保存k个元素。堆的比较方式为元素的频率大小。
  3. 遍历哈希表，将每个元素及其频率加入堆中。
  4. 如果堆中的元素数量超过k，则弹出堆顶元素。
  5. 遍历完成后，堆中剩下的k个元素即为出现频率前k高的元素。

因为要把出现频率低的元素不断从堆顶弹出，故使用小顶堆。
