1，关键字this使用

在 Java 中，this 是一个关键字，它代表当前对象的引用。当你在一个对象内部使用 this 关键字时，你可以引用该对象的所有成员，包括方法和变量。

this 关键字可以用于以下场景：

  1. 当局部变量的名称与实例变量的名称相同时，可以使用 this 来引用实例变量，以区分两者。
  2. 可以在构造器中使用 this() 调用该类的其他构造器，以避免重复代码。
  3. 可以将当前对象作为参数传递给其他方法，例如事件处理器或回调函数。

3，synchronized关键字

synchronized 是 Java 中用于实现同步的关键字。它可用于修饰方法和代码块，实现对类和对象的同步。在多线程环境中，同步是为了保证多个线程能够按照一定的规则对共享资源进行访问。

在修饰方法时，synchronized 关键字会对整个方法体进行同步，即同一时间只有一个线程可以访问该方法。在修饰代码块时，synchronized 关键字会对指定的对象进行同步，即同一时间只有一个线程可以访问指定对象中被 synchronized 关键字修饰的代码块。

Java 中的每⼀个对象都可以作为锁，有三种加锁的⽅式：

（1）对于普通同步⽅法，锁是当前实例对象。

（2）对于静态同步⽅法，锁是当前类的 Class 对象。

（3）对于同步⽅法块，锁是 Synchonized 括号⾥配置的对象。

2，高并发

1，乐观锁和悲观锁

乐观锁和悲观锁是并发控制的两种策略。

悲观锁假定数据随时会被其他线程修改，因此在操作数据时先将其锁定，直到操作完成后才释放锁。悲观锁在高并发下会导致大量线程的阻塞和等待，因此效率较低。

乐观锁则假定数据在并发访问时不会被修改，因此不需要加锁，而是在操作数据时进行比较，如果发现数据已经被其他线程修改，则放弃本次操作并重试。乐观锁的实现方式比较多样化，常用的方式包括版本号、时间戳等。
